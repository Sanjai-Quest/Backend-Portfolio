\# 1. Project Vision



Backend Runtime is not a portfolio website.



Backend Runtime is an interactive software product that introduces its creator through backend engineering concepts.



Instead of reading paragraphs describing skills, visitors should experience how backend systems operate by interacting with the website.



The application itself becomes the demonstration.



Every navigation represents request routing.



Every page represents an individual backend service.



Every animation represents a backend process.



Every interaction teaches an engineering concept without requiring documentation or tutorials.



The portfolio should communicate engineering ability through interaction instead of explanation.



The ultimate objective is for technical recruiters and senior engineers to conclude the following before opening any project.



"He understands distributed systems."



"He thinks in backend architecture."



"He builds production-ready Spring Boot applications."



"I forgot I was browsing a portfolio."



\---



\# 2. Product Philosophy



Traditional portfolios attempt to convince visitors by making claims.



Examples include:



"I know Java."



"I know Spring Boot."



"I know Redis."



"I know Kafka."



This project intentionally avoids that approach.



Visitors should infer those skills naturally.



Instead of saying "I know Redis", the visitor experiences a cache hit.



Instead of saying "I know Kafka", the visitor watches an event move through an event pipeline.



Instead of saying "I understand API Gateways", navigation itself behaves as an API Gateway.



The portfolio never attempts to teach through documentation.



The portfolio teaches through interaction.



Every visual decision exists because it communicates a backend concept.



Decorative animations are forbidden.
